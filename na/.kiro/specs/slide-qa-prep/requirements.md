# Requirements Document

## Introduction

The Slide Q&A Prep feature extends the existing Flask (FastAPI) web application to help slide deck owners prepare for Q&A sessions. The system reads markdown slide content from `slides/<project_name>/md/`, uses an LLM to generate interview-style questions, allows the user to record or upload voice responses to individual questions, transcribes those responses using the existing transcription service, and then uses the LLM to provide feedback on each response based on the actual slide content.

## Glossary

- **QA_Prep_System**: The new feature module within the web application responsible for generating questions, managing voice responses, and providing feedback for slide deck Q&A preparation.
- **Slide_Deck**: A collection of numbered markdown files located at `slides/<project_name>/md/` representing OCR'd slide content.
- **Question_Set**: An ordered list of interview-style questions generated by the LLM from a Slide_Deck's content.
- **Voice_Response**: An audio file uploaded by the user as their spoken answer to a specific question in the Question_Set.
- **Transcription_Service**: The existing Soniox-based transcription module (`app/services/transcription.py` and `transcribe_meetings.py`) that converts audio files to text.
- **LLM_Service**: The OpenAI-compatible API endpoint used for generating questions and feedback, configured via `OPENAI_COMPATIBLE_URL`, `OPENAI_KEY`, and `MODEL` environment variables.
- **Feedback_Report**: The LLM-generated evaluation of a transcribed Voice_Response, assessed against the actual Slide_Deck content.
- **Project_Name**: The folder name under `slides/` that identifies a specific slide deck (e.g., "general", "var-4th-draft").

## Requirements

### Requirement 1: Slide Deck Selection

**User Story:** As a slide deck owner, I want to browse and select a slide deck project from the available decks, so that I can start preparing for a Q&A session on that specific deck.

#### Acceptance Criteria

1. WHEN the user navigates to the Q&A Prep page, THE QA_Prep_System SHALL list all available Project_Name entries found under the `slides/` directory that contain a `md/` subdirectory.
2. WHEN the user selects a Project_Name, THE QA_Prep_System SHALL read all markdown files from `slides/<Project_Name>/md/` and aggregate the content as the Slide_Deck context.
3. IF the selected Project_Name has no markdown files in the `md/` subdirectory, THEN THE QA_Prep_System SHALL display an error message indicating the deck has no slide content.
4. THE QA_Prep_System SHALL require authentication before allowing access to the Q&A Prep page, consistent with the existing application authentication mechanism.

### Requirement 2: Question Generation

**User Story:** As a slide deck owner, I want the system to generate a set of interview-style Q&A questions from my slide deck content, so that I can practice answering likely questions about my presentation.

#### Acceptance Criteria

1. WHEN the user requests question generation for a selected Slide_Deck, THE QA_Prep_System SHALL send the aggregated slide markdown content to the LLM_Service with a prompt instructing generation of interview-style questions.
2. THE QA_Prep_System SHALL generate between 5 and 15 questions per Slide_Deck, covering key topics, data points, and strategic decisions found in the slides.
3. WHEN the LLM_Service returns the Question_Set, THE QA_Prep_System SHALL persist the Question_Set to disk at `output/slide-qa-prep/<Project_Name>/questions.json` so that the questions survive page reloads.
4. WHEN the user views a previously generated Question_Set, THE QA_Prep_System SHALL load the questions from the persisted file rather than regenerating them.
5. THE QA_Prep_System SHALL provide an option for the user to regenerate the Question_Set, replacing the previously stored questions.
6. IF the LLM_Service call fails, THEN THE QA_Prep_System SHALL display an error message to the user with the failure reason.

### Requirement 3: Question Display and Response Upload

**User Story:** As a slide deck owner, I want to see the generated questions and upload my voice response for each question, so that I can practice and get feedback on my answers.

#### Acceptance Criteria

1. WHEN a Question_Set has been generated, THE QA_Prep_System SHALL display all questions in a numbered list on the Q&A session page.
2. THE QA_Prep_System SHALL provide a file upload control next to each question, allowing the user to upload a Voice_Response audio file for that specific question.
3. THE QA_Prep_System SHALL accept audio files with the same extensions supported by the existing application: .m4a, .mp3, .mp4, .wav, .flac, .ogg, .aac, .webm, .amr, .aiff, .asf.
4. WHEN the user uploads a Voice_Response, THE QA_Prep_System SHALL save the audio file to `output/slide-qa-prep/<Project_Name>/responses/q<question_number>_<random_3_char>.<original_extension>`, where `<random_3_char>` is a randomly generated 3-character alphanumeric suffix, allowing multiple attempts per question.
5. IF the user uploads a file with an unsupported extension, THEN THE QA_Prep_System SHALL reject the upload and display an error message indicating the allowed audio formats.
6. WHEN one or more Voice_Responses have been uploaded for a question, THE QA_Prep_System SHALL display all attempts for that question, ordered by upload time, with their respective statuses.

### Requirement 4: Voice Response Transcription

**User Story:** As a slide deck owner, I want my uploaded voice responses to be automatically transcribed, so that the system can analyze the text content of my answers.

#### Acceptance Criteria

1. WHEN a Voice_Response audio file is uploaded for a question, THE QA_Prep_System SHALL invoke the Transcription_Service to convert the audio to text.
2. WHEN transcription completes, THE QA_Prep_System SHALL save the transcript to `output/slide-qa-prep/<Project_Name>/responses/q<question_number>_<random_3_char>.txt`, using the same `<random_3_char>` suffix as the corresponding audio file.
3. WHEN transcripts are available for a question, THE QA_Prep_System SHALL display all transcribed attempts to the user alongside the corresponding question.
4. IF the Transcription_Service fails, THEN THE QA_Prep_System SHALL display an error message to the user indicating the transcription failure reason.

### Requirement 5: LLM Feedback on Responses

**User Story:** As a slide deck owner, I want to receive feedback on my transcribed responses based on the actual slide content, so that I can improve the accuracy and completeness of my answers.

#### Acceptance Criteria

1. WHEN a Voice_Response has been transcribed for a question, THE QA_Prep_System SHALL send the question, the transcribed response, and the original Slide_Deck content to the LLM_Service for evaluation.
2. THE QA_Prep_System SHALL instruct the LLM_Service to evaluate the response for accuracy against the slide content, completeness of key points covered, and clarity of the answer.
3. WHEN the LLM_Service returns the Feedback_Report, THE QA_Prep_System SHALL persist the feedback to `output/slide-qa-prep/<Project_Name>/responses/q<question_number>_<random_3_char>_feedback.md`, using the same `<random_3_char>` suffix as the corresponding audio and transcript files.
4. THE QA_Prep_System SHALL display the Feedback_Report to the user alongside the question and the transcribed response.
5. IF the LLM_Service feedback call fails, THEN THE QA_Prep_System SHALL display an error message to the user with the failure reason.

### Requirement 6: Session Overview and Progress Tracking

**User Story:** As a slide deck owner, I want to see an overview of my Q&A preparation progress, so that I can track which questions I have answered and received feedback on.

#### Acceptance Criteria

1. THE QA_Prep_System SHALL display a progress indicator showing the count of questions answered out of the total Question_Set size.
2. WHEN the user views the Q&A session page, THE QA_Prep_System SHALL visually distinguish questions that have a Voice_Response uploaded, questions that have been transcribed, and questions that have received a Feedback_Report.
3. THE QA_Prep_System SHALL allow the user to upload additional Voice_Response attempts for any question without removing previous attempts.

### Requirement 7: Navigation and Integration

**User Story:** As a user of the web application, I want to access the Q&A Prep feature from the main navigation, so that I can easily find and use the feature alongside existing functionality.

#### Acceptance Criteria

1. THE QA_Prep_System SHALL be accessible via a dedicated route at `/qa-prep` for the deck selection page.
2. WHEN a Slide_Deck is selected, THE QA_Prep_System SHALL serve the Q&A session page at `/qa-prep/<Project_Name>`.
3. THE QA_Prep_System SHALL include a navigation link in the application home page or base template to access the Q&A Prep feature.
4. THE QA_Prep_System SHALL follow the existing application patterns for route definitions, template rendering, and authentication.

### Requirement 8: Docker Compose Volume Mapping

**User Story:** As a developer deploying the application, I want the `slides` folder to be mounted into the Docker container, so that the Q&A Prep feature can access slide deck content at runtime.

#### Acceptance Criteria

1. THE `docker-compose.yml` at `/home/ubuntu/nspaceresearch/na/docker-compose.yml` SHALL include a volume mapping for the `slides` directory (e.g., `./slides:/app/slides`) in the `web` service, following the same pattern as the existing `./meeting-audios:/app/meeting-audios` mapping.
2. THE `docker-compose.yml` SHALL also include a volume mapping for the `output` directory (e.g., `./output:/app/output`) so that generated questions, responses, and feedback persist outside the container.
