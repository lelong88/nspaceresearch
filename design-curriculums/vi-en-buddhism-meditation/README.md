## Recreation Context

### Method

Each curriculum was created via a standalone Python script (`create_<topic>.py`) containing hand-crafted Vietnamese/English content. An orchestrator script (`orchestrator.py`) created the collection, 3 series, wired them together, and set series display orders. Each curriculum script:

1. Built the content JSON with hand-written text (introAudio, reading passages, descriptions, writing prompts)
2. Validated content via `validate_content.py` (format-aware validator checking activity sequences, strip keys, structural rules)
3. Called `curriculum/create` with `language="en"`, `userLanguage="vi"`
4. Added the curriculum to its series via `curriculum-series/addCurriculum`
5. Set display order via `curriculum/setDisplayOrder`
6. Set price to 49 via `curriculum/setPrice`

### Authentication

All API calls used Firebase ID token for UID `zs5AMpVfqkcfDf8CJ9qrXdH58d73` via shared `firebase_token.py`.

### API Base URL

`https://helloapi.step.is`

### Key Design Decisions

- **Three skill focus formats**: story_reading (narrative passages, contentTypeTags: ["story"]), speaking (oral production emphasis, contentTypeTags: []), balanced (all skills, contentTypeTags: [])
- **Private by default**: No `setPublic` calls — curriculums need content generation (audio, illustrations) before being exposed to learners
- **Bilingual content**: Vietnamese for user-facing text (title, description, preview), English for reading passages, bilingual for introAudio scripts
- **Tone variety**: Pre-planned description and farewell tone assignments satisfying adjacency and distribution constraints
- **No template content**: Every piece of learner-facing text individually crafted per curriculum
