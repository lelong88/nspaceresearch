## Creation Method

Each curriculum was created via a standalone Python script calling `POST https://helloapi.step.is/curriculum/create` with:
- `language: "en"` (target language)
- `userLanguage: "vi"` (learner language)
- `content`: hand-written JSON with title, description, preview, learning sessions, and activities

All learner-facing text (descriptions, introAudio scripts, reading passages, writing prompts) was individually crafted per curriculum — no templates or string interpolation for content.

Shared `validate_curriculum.py` helper handled structural validation only (activity schema, session counts, word counts, forbidden activities).

Series and collections were created via batch orchestrator scripts (`create_batch_XX_series.py`) that called:
- `curriculum-series/create` — create series
- `curriculum-series/addCurriculum` — add curriculums to series
- `curriculum/setDisplayOrder` — set display order per curriculum
- `curriculum-series/setDisplayOrder` — set series display order
- `curriculum-collection/create` — create collections (when new)
- `curriculum-collection/addSeriesToCollection` — wire series to collections

All Python scripts were deleted after verification. Only README.md files remain in each batch folder.

## Recreation Context

To recreate any batch:
1. Read the batch README in `batch-XX-*/README.md` for curriculum IDs, series IDs, and collection IDs
2. Write standalone Python scripts following the spec in `.kiro/specs/vi-en-curriculum-expansion/`
3. Each script needs hand-written content, structural validation, and API call to `curriculum/create`
4. Run an orchestrator to create series, wire curriculums, set display orders, and add to collections
5. Verify via SQL queries in the batch README
6. Delete scripts after verification

The full spec (requirements, design, tasks) is in `.kiro/specs/vi-en-curriculum-expansion/`.
