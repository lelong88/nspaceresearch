# Tiếng Hàn Cho Người Mới Bắt Đầu (Vietnamese-Korean Beginner Curriculums)

20 Korean-learning curriculums for Vietnamese-speaking adults at the beginner level, organized into 1 collection and 3 series.

- **Language pair**: `userLanguage="vi"`, `language="ko"` (Vietnamese speakers learning Korean)
- **Audience**: Vietnamese adults beginning their Korean learning journey
- **Method**: Python scripts via `curriculum/create` API (scripts deleted after creation)
- **Script directory**: `vi-ko-beginner-curriculums/`
- **Vocabulary writing system**: Hangul only (no Hanja); Revised Romanization used in introAudio for pronunciation guidance

## Recreation Instructions

Source materials (per-curriculum `create_*.py` scripts, `orchestrator.py`, `validate_content.py`) are deleted after successful import to the database. Full content for any curriculum is recoverable from the DB via `curriculum/getOne` or MCP postgres queries on the `curriculum` table.

Original creation method (documented in `.kiro/specs/vi-ko-beginner-curriculums/`):

- One standalone Python script per curriculum, hand-crafted Korean (Hangul, no Hanja) and Vietnamese marketing text
- One orchestrator script (`orchestrator.py`) created the collection and 3 series and wired them together
- A beginner-specific `validate_content.py` validated each content JSON before upload (NO lowercase enforcement for Korean Hangul; format-aware checks for `beginner_mini` vs `beginner_short`)
- Shared root-level `api_helpers.py` handled all API calls
- Auth via `firebase_token.get_firebase_id_token('zs5AMpVfqkcfDf8CJ9qrXdH58d73')`
- API base: `https://helloapi.step.is`

To recreate from scratch (the script directory is empty after cleanup):

1. **Run `orchestrator.py` first** — creates the 1 collection (`hzhgym0f`), the 3 series (`7iyamk2u`, `ue9ur79g`, `3cxa5osm`), wires the series into the collection, and sets each series display order (1, 2, 3). Logs all IDs to stdout. Without this step the curriculum scripts have no series to attach to.
2. **Update each `create_*.py` script with the new series ID** — paste the appropriate `SERIES_ID` constant at the top of each script (Series 1 ID for the 8 mini scripts, Series 2 ID for `create_food.py`/`create_shopping.py`/`create_transport.py`/`create_hotel.py`/`create_office.py`/`create_seasons.py`, Series 3 ID for `create_hobbies.py`/`create_health.py`/`create_housing.py`/`create_culture.py`/`create_technology.py`/`create_sports.py`).
3. **Run the 8 Series 1 scripts in display-order** — `create_greetings.py`, `create_numbers.py`, `create_family.py`, `create_colors.py`, `create_weather.py`, `create_drinks.py`, `create_days.py`, `create_emotions.py`. Each script validates content via `validate(content, "beginner_mini")`, calls `curriculum/create`, then `curriculum-series/addCurriculum`, then `curriculum/setDisplayOrder`, then `curriculum/setPrice` (price 9).
4. **Run the 6 Series 2 scripts in display-order** — `create_food.py`, `create_shopping.py`, `create_transport.py`, `create_hotel.py`, `create_office.py`, `create_seasons.py`. Validates against `beginner_short` format. Price 19.
5. **Run the 6 Series 3 scripts in display-order** — `create_hobbies.py`, `create_health.py`, `create_housing.py`, `create_culture.py`, `create_technology.py`, `create_sports.py`. Validates against `beginner_short` format. Price 19.
6. **Do NOT call `curriculum/setPublic`** — these curriculums remain private until content generation (audio, illustrations) completes.

To recreate a single curriculum from existing DB content:

1. **Retrieve content from DB** — `curriculum/getOne` API or `SELECT content FROM curriculum WHERE id = '<id>'`
2. **Strip generated keys** — remove `mp3Url`, `illustrationSet`, `chapterBookmarks`, `segments`, `whiteboardItems`, `userReadingId`, `lessonUniqueId`, `curriculumTags`, `taskId`, `imageId` from the content JSON before reuploading
3. **Create new curriculum** — `curriculum/create` with top-level body fields `language="ko"`, `userLanguage="vi"`, and `content` (JSON string)
4. **Add to series** — `curriculum-series/addCurriculum` (or `curriculum-collection/addCurriculumToSeries`) with the appropriate series ID
5. **Set display order** — `curriculum/setDisplayOrder` with the curriculum ID and order from the table above
6. **Set price** — `curriculum/setPrice` with `price=9` (mini) or `price=19` (short)
7. **Replace existing** — to swap a curriculum, first `curriculum-series/removeCurriculum`, then `curriculum/delete`, then create the replacement
