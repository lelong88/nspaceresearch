## Concept

An **evergreen, AI-generated coaching campaign** where every user receives a completely personalized series of emails and in-app banners. Unlike bulk campaigns (one template → many recipients), this campaign produces **unique content per user** based on their learning data.

The goal: deliver the right educational content, at the right time, in the right channel (email vs. in-app banner) — acting as a personal learning coach.

---

## How It Differs from Standard Campaigns

| Aspect | Standard bulk campaign | Coaching campaign |
|--------|----------------------|-------------------|
| Content | One `body.html` template for all | Unique HTML per user, generated by AI |
| Cadence | One-shot or scheduled | Ongoing / evergreen — triggered by user behavior |
| Personalization | `{{variables}}` substitution | Fully AI-written based on learning data |
| Storage | `campaigns/{slug}/body.html` | `campaigns/coaching/{user_uid}/` |
| Scale | 1 template → N recipients | N templates → N recipients |

---

## Data Sources for Personalization

The AI agent has access to rich per-user learning data:

| Table | Key signals |
|-------|-------------|
| `engage_user_curriculum` | Curricula engaged, progress %, completion, scores (writing sentences/paragraphs, read-alongs), first/last activity timestamps |
| `user_streak` | Current daily streak, longest streak, last active date |
| `user_activity` | Per-day activity records |
| `user_session` | Speaking sessions with fluency, prosody, accuracy, content scores + pronunciation feedback |
| `my_user_lesson` | Lessons completed |
| `my_user_curriculum_completion` | Curriculum completion records |
| `my_user` | User profile (name, email, timezone offset) |

## Active Channel: In-App Banners Only

> ⚠️ **Email is disabled** until DMARC/deliverability setup is complete. All coaching touchpoints are delivered via in-app banners for now.

Potential coaching touchpoints, selected dynamically per user:

- **Learning statistics summary** — weekly/monthly progress highlights
- **Streak encouragement** — celebrate milestones or nudge after inactivity
- **Skill diagnosis** — highlight weak areas (e.g., low writing pass rate, pronunciation gaps)
- **Curriculum recommendations** — suggest next curriculum based on completed ones
- **Completion celebration** — congratulate curriculum completion
- **Re-engagement** — reach out after days of inactivity
- **Learning tips** — contextual advice based on current activity patterns

- MCP postgres (read-only SQL) and the Cloudflare Worker API endpoints are the primary interfaces for the AI agent
